# 연산자

![](../.gitbook/assets/operator.png)

## 🤔 연산자란 무엇인가요?

우리는 대한민국의 의무교육을 받으며 수학이란 과목을 공부 했었습니다. \( 저는 수포자 출신입니다... 😅  \)  
수학에서는 특정 연산자를 통해 값을 도출하고, 그 결과를 얻어냈습니다.

프로그래밍 언어에서도 마찬가지로, 수학과 동일하게 특정 연산자를 가지고 있어서 그 연산자를 활용해 각 연산이 될 대상을 계산하여 값을 도출하게 됩니다.  
여기서 각 연산이 될 대상을 **피연산자** 라고 표현합니다. 

프로그래밍 언어에는 정말 많은 연산자가 있지만, 자바스크립트만이 가지는 연산자의 특징이 있기도 합니다.  
기본적인 것 부터 자주 사용하는 연산자를 위주로 살펴보도록 하겠습니다.

## 📚 연산자의 종류에는 무엇이 있나요?

### 1. 산술 연산자

산술 연산자는 우리가 가장 흔하게 볼 수 있고 일상생활에서 가장 자주 사용하는 연산일 것입니다.  
먼저 우리가 가장 자주 사용하는 사칙연산을 통해 연산하는 방법을 알아보겠습니다.

```javascript
9 + 1 // 10;
5 - 1 // 4;
3 * 2 // 6;
5 / 2 // 2.5;
7 % 3 // 1;
```

딱 봐도 딱히 설명할 필요성을 느끼지 못할 정도인 연산자들의 집합입니다. 이 연산자들은 우리는 **이항 산술 연산자** 라고 부릅니다. \( 딱히 용어가 중요하진 않습니다. \)

하지만 산술 연산자에서도 정말 중요한 복병인 친구들이 있습니다.  
바로 **단항 산술 연산자** 라고 불리우는 녀석들인데요, 한 번 살펴보겠습니다.

| 단항 산술 연산자 | 의미 |
| :--- | :--- |
| x++ | 후 증가 |
| ++x | 선 증가 |
| x-- | 후 감소 |
| --x | 선 감소 |

단항 산술 연산자에는 4가지의 형태가 있습니다.  
연산자 형태만 봐서는 이해가 잘 되지 않으니 직접 코드를 보겠습니다.

```javascript
var count = 1;

count++; // count = count + 1;
console.log(count); // 2

count--; // count = count - 1;
console.log(count); // 1
```

위 코드를 보면 도대체 이게 왜... 🤔  라는 생각이 드실 수 있습니다.  
하지만 단순하게 ++ 혹은 -- 라는 연산자를 통해서 count라는 변수의 값을 **증가 혹은 감소**시키는 역할을 해줍니다.

위의 두 코드만 보면 정말 간단해 보이지만 추가로 2개가 들어가면 조금 헷갈려집니다.

```javascript
var count = 1;
var result = 1;

result = count++;
console.log(count); // 2
console.log(result); // 1

result = ++count;
console.log(count); // 2
console.log(result); // 2

result = count--;
console.log(count); // 0
console.log(result); // 1

result = --count;
console.log(count); // 0
console.log(result); // 0
```

뭔가 느낌이 오시나요..?  
눈치 채셨을지는 모르겠지만 두개 연산의 차이점은 바로 **연산 순서**를 의미합니다.

변수 뒤에 위치한 단일 산술 연산자는 대입한 결과에 **먼저 반영되지 않고** **이후에 연산 처리**가 되어 연산한 변수 본인만 변동이 되어있게 되고, 변수 앞에 위치한 단일 산술 연산자는 **먼저 연산이 일어난 다음 대입**이 되어 결과에 반영이 된 모습입니다.

### 2. 할당 연산자

할당 연산자는 피연산자의 값을 변수에 할당해주는 역할을 합니다.  
설명만 들으면 이해가 가기 쉽지 않은 개념이나 직접 보면 충분히 이해가 가능합니다.

| 할당 연산자 | 예 | 동일 표현 |
| :--- | :--- | :--- |
| = | x = 5 | x = 5 |
| += | x += 5 | x = x + 5 |
| -= | x -= 5 | x = x - 5 |
| \*= | x \*= 5 | x = x \* 5 |
| /= | x /= 5 | x = x / 5 |
| %= | x %= 5 | x = x % 5 |

```javascript
var x;

x = 10;
console.log(x); // 10

x += 5; // x = x + 5;
console.log(x) // 15

x -= 5; // x = x - 5;
console.log(x) // 10
```

위의 표에서 정리된 것처럼 특정 연산자와 '**='** 기호를 결합하면 동일 표현과 같은 식으로 완성됩니다.  
처음에는 조금 어색하여 코드를 분석하거나 작성하기 어려울 수 있지만, 추후 조금씩 익숙해지면 정말 편리하게 사용할 수 있는 연산자입니다.  
주로 반복되는 **특정 값을 누적**하거나, **계산** 해야하는 부분에서 자주 사용됩니다.

### 3. 비교 연산자

비교 연산자는 좌, 우 값을 서로 비교한 다음 **그 결과를 불리언 값**으로 나타내줍니다.  
즉, 참과 거짓을 판단하기 때문에 보통 특정 조건을 만들고 그 조건에 만족하는 코드를 작성할 때 주로 사용됩니다.

| 비교 연산자 | 의미 | 예 | 설명 |
| :--- | :--- | :--- | :--- |
| == | 동등 비교 | x == y |  x와 y 값이 같음 |
| === | 일치 비교 | x === y | x와 y의 값, 타입이 같음 |
| != | 부동등 비교 | x != | x와 y 값이 다름 |
| !== | 불일치 비교 | x !== y | x와 y의 값, 타입이 다름 |
| &gt; | 대소 비교 | x &gt; y | x가 y보다 크다. |
| &lt; | 대소 비교 | x &lt; y | x가 y보다 작다. |
| &gt;= | 대소 비교 | x &gt;= | x가 y보다 크거다 작다. |
| &lt;= | 대소 비교 | x &lt;= | x가 y보다 작거나 같다. |

```javascript
// 동등 비교
1 == 1 // true
1 == '1' // true

// 일치 비교
1 === 1 // true
1 === '1' // false

// 부동등 비교
1 != 2 // true
1 != '2' // true

// 불일치 비교
1 !== 2 // true
1 !== '2' // false

// 대소 비교
1 > 0 // true
1 < 0 // false
1 >= 1 // true
2 <= 1 // false
```

간략이 작성한 위 비교들을 보면, 단지 값만을 비교할 때는 동등 비교를 사용하여 비교한다면 딱히 크게 값에 대한 타입을 고려하지 않아도 비교가 되기 때문에 더 좋아 보일 수도 있습니다.  
벗 하지만❗️ 값을 비교 할 때 타입을 고려하지 않고 비교를 하다보면 의도치 않은 결과를 받게 될 수도 있고, 그로 인해 정상 작동하던 애플리케이션이 비정상적으로 동작하게 될 수도 있습니다.

타입을 정확하게 비교하여 의도한대로 동작하도록 애플리케이션을 개발하는 것도 개발자의 의무이기 때문에, 타입까지 완전하게 비교하는 **일치 비교**를 선호하는 것이 좋습니다. 👌

### 4. 삼항 연산자

삼항 연산자는 특정 조건에 대하여 반환된 불리언 값을 **?** 기호를 기준으로 좌, 우 피연산자를 실행하도록 하는 연산자입니다.  
주로 간단한 조건문에 많이 사용되며 복잡하지 않을 경우 코드가 깔끔해지고, 가독성이 좋아지기 때문에 자주 사용하는 패턴입니다.

```javascript
var nickname = 'tigger';

var result = nickname === 'tigger' ? "내 닉네임" : "다른 닉네임"
console.log(result); // 내 닉네임
```

 **?**를 기점으로 좌측은 조건식, 우측은 **:**를 기점으로 true : false 결과에 따른 피연산자들입니다.  
조건이 true라면, :의 좌측의 피연산자가 할당되고, false라면 우측의 피연산자가 할당되게 됩니다.

### 5. 논리 연산자

우항과 좌항의 피연산자들을 논리에 맞게 판단하는 연산입니다.  
특정 **조건**에 대한 코드를 작성할 때 정말 많이 사용하는 방법입니다. 또한 **암묵적 타입변화**를 통해서 **단축 평가**라는 부분에 엄청난 활용도가 있으니 잘 이해하신다면 큰 도움이 됩니다.

| 논리 연산자 | 의미 |
| :--- | :--- |
| \|\| | 논리합\(OR\) |
| && | 논리곱\(AND\) |
| ! | 부정\(NOT\) |

```javascript
// ||(OR) 연산자
true || true; // true
true || false; // true 
false || true; // true
false || false; // false

// &&(AND) 연산자
true || true; // true
true || false; // false
false || true; // false
false || false; // false

// !(NOT) 연산자
!true; // false
!false; // true
```

코드를 보시면 두 피연산자 중 연산자에 따라 나타내는 값이 달라집니다.  
**OR**의 경우에는 **둘 중 하나**만이라도 true라면 true가 나타나고, **AND**의 경우에는 **둘 다** true여야만 true가 나타납니다.

### 6. typeof 연산자

typeof 연산자는 피연산자의 **데이터 타입**을 **문자열**로 나타내줍니다.  
자바스크립트는 선언시 타입을 정하지 않기 때문에 초기화가 일어난 후 할당된 이후 타입 체크를 하는 방법이 굉장히 한정적입니다. \( 물론 최근에는 Typescript를 통해 개발시 타입을 함께 선언하며 개발이 가능합니다. \)

그래서 typeof 연산자를 통해 타입을 체크하여 안전한 타입을 유지하며 개발하는데에 주로 사용됩니다.  
여기서 유추할 수 있듯이 데이터 타입을 나타내준다는 것은 자바스크립트의 데이터 타입 7개를 나타내 준다는 것이겠죠?

하지만 **충.격.적.**이게도 아닙니다.. 😱  
코드를 보면 더욱 충격적입니다...;;;

```javascript
typeof "tigger" // "string"
typeof 0 // "number"
typeof true // "boolean"
typeof undefined // "undefined"
typeof Symbol() // "symbol"
typeof null // "object"
typeof [] // "object"
typeof {} // "object"
typeof function // "function"

var tigger = null;
typeof tigger === null; // false
tigger === null; // true
```

이게 무슨...;;; 분명 자바스크립트 타입은 7개라고 해놓고서.. 갑자기 typeof로 확인했더니 7개가 아니랍니다...  
여기서 주의깊게 봐야할 부분은 바로 **null**과 **function**입니다.  
분명 function은 object고 null은 null 이라고 하셨잖아요 ㅠㅠㅠ...

**null이 object**인 부분은 자바스크립트를 최초에 만든 후 **첫 번째 버전**을 릴리즈 했을 때의 오류인데, 그 첫 번째 버전을 기준으로 개발한 서비스들이 존재하기 때문에 현재 까지도 수정되지 않은 레전드입니다.

이 부분을 주의해서 타 프로그래밍 언어와는 다르게 자바스크립트는 null 타입을 체크할 때 typeof 연산자를 사용하는 것이 아니라 일치 비교\(===\) 를 통해서 null을 비교해야 정상적인 값을 도출할 수 있습니다.
